# 摘要:

透過找到陰影變數矯正推薦系統的偏差。

# 問題: 

為了消除推薦系統的偏差，最好的數據蒐集方法是進行隨機實驗。但為了不影響系統運行等現實因素，本研究使用觀測數據進行推理。

偏差的主要來源有兩個:選擇偏差及混淆偏差

==使用者評分==通常具有==missing not at random 非隨機缺失特性==，意旨用戶通常只會對極度喜歡或極度討厭的物品進行評論，所以根據使用者評論訓練出的模型會具有偏見。

協同過濾模型藉由分析使用者特徵與物品互動紀錄來預測使用者對未見過物品偏好，但由於使用者評分數據的非隨機性缺失偏差使得模型無法有效預測，陰影變量是一種特殊的協變量，可以做為混淆因子的代理變量，藉由==找到陰影變量並加入模型即可阻斷混淆路徑==；==建構偏差較小模型==。

# 解決辦法:
作者提出藉由Shadow Variable Catcher 找到 shadow variable陰影變量，並建構了一個新的框架Shadow Debiased Recommender，此架構在協同模型訓練中加入陰影變量以消除模型的選擇性偏差。


# 名詞解釋:

1. 協變量 (Covariate) : 協變量是指在研究中被測量或觀測到的輸入變量或特徵。它們是除了核心的「處理」(曝光)和「結果」(評分)之外，可能對結果產生影響的任何其他變量，例如: 用戶特徵、物品特徵等。

2. 混淆因子(Confounder): 混淆因子是一個潛在的、未觀測到的用戶特質，它同時影響了用戶選擇去看/評分某個物品的行為，以及用戶對該物品的真實喜愛程度。混淆因子會建構混淆路徑導致關係建構錯誤(舉例: 喜歡物品的人更容易選擇評分，他們的真實評分也會偏高，不喜歡的人則是不會參與評分導致評分缺失)

3. 對撞機(Collider): 若一個變量是多個其他變量的影響結果，在對該變量進行控制等條件判斷時，影響該便葬的其他獨立變量會產生虛假的相關性。

技術:
Collaborative Filtering 協同過濾:
- Memory based:
	1. User-Based: 找到與用戶具有相似偏好的其他用戶，根據他們對類似物品的評分預測使用者的可能評分。
	2. Item-based: 找到相似的其他物品，根據使用者對其的評分進行預測。
- Model Based:
	1. 矩陣分解: 透過構建用戶特徵矩陣與物品特徵矩陣內積計算評分。
	2. 機器學習: 神經協同過濾 (Neural Collaborative Filtering, NCF) 用多層感知機 (MLP) 取代傳統 MF 中的簡單內積操作，來學習用戶和物品潛在向量之間更複雜的、非線性的互動函數。

| **手段 (Adjustment Strategy)**             | **目的**                                            | **適用條件/前提**                                        | **與偏差的關係**               |
| ---------------------------------------- | ------------------------------------------------- | -------------------------------------------------- | ------------------------ |
| **1. 後門調整 (Backdoor Adjustment)**        | 消除**可觀測到的混淆因子**造成的偏差。                             | **前提：** **所有**混淆因子 $L$ 都必須是**可觀測**到的（即滿足後門準則）。     | 消除**混淆偏差**。              |
| **2. 工具變量 (Instrumental Variables, IV)** | 消除**未觀測到的混淆因子**造成的偏差。                             | **前提：** 必須找到一個滿足**三個嚴格假設**（相關性、排除限制、外生性）的工具變量 $Z$。 | 消除**混淆偏差**。              |
| **3. 前門調整 (Frontdoor Adjustment)**       | 消除**未觀測到的混淆因子**造成的偏差。                             | **前提：** 必須找到一個滿足**三個嚴格準則**（特別是完全中介）的**中介變量 $M$**。  | 消除**混淆偏差**。              |
| **4. 陰影變量 (Shadow Variables, SVC/SDR)**  | 消除**未觀測混淆因子**造成的偏差，特別適用於 **MNAR 數據**（如推薦系統的選擇偏差）。 | **前提：** 必須找到或學習一個滿足**條件獨立性**假設的潛在代理變量 $Z$。         | 解決**選擇偏差** (由未觀測混淆因子引起)。 |
